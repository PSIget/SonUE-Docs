# Markdown

## Базові поняття

Ця інформація коротко переказує інформацію з сайту [Markdown Guide](https://www.markdownguide.org/basic-syntax/)

### Заголовки

Щоб створити заголовок, додайте перед словом або фразою знаки номерів `#`. Кількість знаків номерів повинна відповідати рівню заголовка. Наприклад, щоб створити заголовок третього рівня `<h3>`, використовуйте три знаки номерів, наприклад, ### Мій заголовок.

```md filename="Markdown"
# Заголовок першого рівня
## Заголовок другого рівня
### Заголовок третього рівня
#### Заголовок четвертого рівня
##### Заголовок п'ятого рівня
###### Заголовок шостого рівня
```

### Акцент
Ви можете додати акцент, зробивши текст напівжирним або курсивним.

#### Напівжирний
- `I just love **bold text**.` - I just love **bold text**.
- `I just love __bold text__.` - I just love __bold text__.
- `Love**is**bold` - Love**is**bold

#### Курсив

- `Italicized text is the *cat's meow*.` - Italicized text is the *cat's meow*.
- `Italicized text is the _cat's meow_.` - Italicized text is the _cat's meow_.
- `A*cat*meow` - A*cat*meow

#### Напівжирний і курсивний

- `This text is ***really important***.` - This text is ***really important***.
- `This text is ___really important___.` - This text is ___really important___.
- `This text is __*really important*__.` - This text is __*really important*__.

### Лінія (HR)

Щоб створити горизонтальну лінійку, використовуйте три або більше зірочок `***`, дефісів `---` або символів підкреслення `___` у окремому рядку.

```md filename="Markdown"
***

---

_________________
```

Візуалізований вивід всіх трьох виглядає однаково:

---

### Посилання
Щоб створити посилання, заключіть текст посилання у квадратні дужки (наприклад, `[Duck Duck Go]`), а потім одразу після цього вкажіть URL-адресу у круглих дужках (наприклад, `(https://duckduckgo.com)`).

```md filename="Markdown"
Мій улюблений пошуковий двигун - це [Duck Duck Go](https://duckduckgo.com "Найкращий пошуковий двигун для приватності").
```

Відображений результат буде виглядати наступним чином:

Мій улюблений пошуковий двигун - це [Duck Duck Go](https://duckduckgo.com "Найкращий пошуковий двигун для приватності").

<Callout type="info" emoji="ℹ️">
    Ви також можете вставити просто посилання без будь-якого форматування.
</Callout>

### Зображення
Щоб додати зображення, використовуйте знак оклику `!`, який замінює текст у дужках, а після цього вкажіть шлях або URL-адресу ресурсу зображення у дужках. За бажанням ви можете додати заголовок в лапках після шляху або URL-адреси.

```md filename="Markdown"
![Заповнювач!](https://placehold.co/832x400/ "Заповнювач")
```

## MDX

Усі ваші файли `.mdx`, що знаходяться в каталозі `pages`, будуть відображатися за допомогою [MDX](https://mdxjs.com/about) - розширеного формату Markdown з підтримкою компонентів React.

Наприклад, ви можете імпортувати та використовувати компоненти React всередині файлів Markdown наступним чином:

```mdx filename="Markdown"
## Привіт, MDX

import { useState } from 'react'

export function Counter({ children }) {
  const [count, setCount] = useState(0)
  return (
    <button onClick={() => setCount(count + 1)}>
      {children}
      {count}
    </button>
  )
}

<Counter>**Кліки**: </Counter>
```

Це буде перетворено на:

import Shadow from './markdown.ru/components/shadow'
import { useState } from 'react'

export function Counter({ children }) {
  const [count, setCount] = useState(0)
  return (
    <button onClick={() => setCount(count + 1)}>
      {children}
      {count}
    </button>
  )
}

<div className="mt-6 rounded border border-gray-200 p-4 dark:border-gray-900">
  <Shadow mode="open">
    <h2>Привіт, MDX</h2>
    <Counter>**Кліки**: </Counter>
  </Shadow>
</div>

Крім базового MDX, документація також включає деякі розширені можливості Markdown.

### Власний ідентифікатор заголовка

Ви можете вказати власний ідентифікатор заголовка, використовуючи формат `## Мій заголовок [#власний-id]`. Наприклад:

```md filename="Markdown"
## Довгий заголовок [#про]
```

У цьому прикладі `#про` буде використовуватися як посилання на заголовок, замінюючи стандартний `#довгий-заголовок`.

### Особливості

#### Вбудований код

Також підтримується вбудоване підсвічування синтаксису, наприклад `let x = 1{:jsx}`, за допомогою синтаксису `{:}`:

```md filename="Markdown"
Inlined syntax highlighting is also supported `let x = 1{:jsx}` via:
```

#### Підсвічування рядків

Ви можете виділити певні рядки коду, додавши атрибут `{}` до блоку коду:

````md filename="Markdown"
```js {1,4-5}
import { useState } from 'react'

function Counter() {
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
}
```
````

Результат:

### З динамічним вмістом

Оскільки підсвічування синтаксису виконується на етапі збірки, ви не можете використовувати динамічний вміст у блоках коду. Однак, оскільки MDX є дуже потужним, є обхідний шлях через клієнтський JS. Наприклад:

import { useEffect, useRef } from 'react'

export function DynamicCode({ children }) {
  const ref = useRef()
  const tokenRef = useRef()
  // Find the corresponding token from the DOM
  useEffect(() => {
    if (ref.current) {
      const token = [...ref.current.querySelectorAll('code span')].find(
        el => el.innerText === '1'
      )
      tokenRef.current = token
    }
  }, [])
  return (
    <>
      <div ref={ref} style={{ marginTop: '1.5rem' }}>
        {children}
      </div>
      <a
        onClick={() => {
          tokenRef.current.innerText =
            (parseInt(tokenRef.current.innerText) || 0) + 1
        }}
        style={{
          cursor: 'pointer',
          userSelect: 'none'
        }}
      >
        Increase the number
      </a>
      <a
        onClick={() => {
          tokenRef.current.innerText = '1 + 1'
        }}
        style={{
          marginLeft: '.5rem',
          cursor: 'pointer',
          userSelect: 'none'
        }}
      >
        Change to `1 + 1`
      </a>
    </>
  )
}

<DynamicCode>
```js filename="dynamic_code.js"
function hello () {
  const x = 2 + 3
  console.log(1)
}
```
</DynamicCode>

Цей обхідний шлях має обмеження: оновлене вміст не буде знову підсвічуватися. Наприклад, якщо ми оновимо число на `1 + 1`, воно буде неправильно підсвічене.
